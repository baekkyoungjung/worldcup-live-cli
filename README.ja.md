# worldcup-live-cli

> **コードを書くようにサッカーを観る。**

[English](README.md) · [한국어](README.ko.md) · [Español](README.es.md) · [Português](README.pt.md) · **日本語**

ワールドカップのライブ中継を **ロガー形式のリアルタイムテキスト**（`log` / `warn` / `error` / `CRITICAL`）に変換し、作業中の Claude Code セッションへそのまま流し込むプラグインです。別画面なし、コンテキストの切り替えなし。遠目にはアプリケーションのログ、近くで読めば実況の絶叫です。

```
14:32:21 [log]      両チームが中盤でボールを回します
14:32:31 🟡 [warn]   スコットランドが右サイドでコーナーキックを獲得
14:33:05 🔴 [error]  ハイチのペナルティエリア内、シュートがゴールを脅かします
14:35:48 🟥 [CRITICAL] ゴール！ John McGinn — HAI 0 : 1 SCO
```

---

## なぜ Claude Code の*中*で動くのか

ここが肝心です。試合はブラウザのタブや別アプリではなく、今コードを書いているまさにそのセッションの中に流れ込みます。

- **コンテキストの切り替えがない。** ターミナル / IDE から離れません。ビルドログやテストランナーのように、中継がセッション内にテキストで現れます。ログを眺めるのと同じ感覚でちらっと見ればいい。
- **スキルが自分のストリームを消費する。** バックグラウンドのデーモンがスコアをポーリングし、スキルがログを `follow` して新しい行だけをセッションに出力します。自分で `tail` を回す必要も、プロセスを見張る必要もありません ―― *「中継を始めて」* と言うだけです。
- **流れを見失わないための recap。** バグ修正で少し席を外した？ **10 分ごと**（およびハーフタイム・給水タイムごと）に、見逃した区間を 2〜4 行で要約して投稿します。戻ってひと目で追いつき、またコードに戻ればいい。
- **無音がない。** 静かな区間でも約 **10 秒ごと**にアンビエントな `[log]` 行を流すので、中継が止まって見えることはありません。ただし **スコア・時間・選手名は常に実データ**で、アンビエント行は試合の流れを描写するだけで事実を捏造しません。
- **おまけのカモフラージュ。** ロガー出力なので、通りすがりの人にはログを tail している開発ツールにしか見えません。（このプロジェクトは元々 *사장 몰래* ―― 「上司に内緒で」という名前で始まりました。その出自は今や見出しではなく一つの機能です。）

## インストール

Claude Code でこのリポジトリをマーケットプレイスとして追加し、プラグインをインストールします:

```
/plugin marketplace add baekkyoungjung/worldcup-live-cli
/plugin install worldcup-live-cli@worldcup-live-cli
```

インストールされるのは単一バンドル（`dist/poll.mjs`）とアセットのみ ―― **`npm install` 不要、ランタイム依存ゼロ。** **Node 18+** さえあれば動きます。実況に使う `claude` CLI が無い環境では、組み込みテンプレートにフォールバックします。

## 使い方

インストール後は、セッションに自然言語で話しかけるだけ ―― デーモンの起動からセッション内ライブ中継、recap の投稿まで、すべてスキルが代行します。

| こう言うと | こう動く |
|---|---|
| 「今日の試合は？」 | 今日の試合一覧と eventId を表示する |
| 「中継を始めて」 / 「試合を観たい」 | ライブの試合を選んでセッションに中継を流す |
| 「前の試合をリプレイ」 / 「ニセモノで見せて」 | 終了した試合を圧縮した疑似ライブで再生する |
| 「スコアどうなってる？」 | 現在のスコアと直近の状況を要約する |
| 「中継を止めて」 | バックグラウンドのデーモンを停止する |

## 中継フォーマット ―― 重大度ロガー

ESPN のイベントを危険度で分類し、prefix を付けます。フィールド上の位置座標が無いため、イベントの種類と ESPN テキストのキーワード（"box"、"corner"、"one-on-one" など）から危険度を近似します。

| prefix | 色 / セッション | 基準 |
|---|---|---|
| `[log]` | 既定 | 中盤の競り合い、通常のファウル、オフサイド、交代など |
| `[warn]` | 黄 / 🟡 | コーナー・フリーキック・セットプレー、危険地域へのパス/クロス |
| `[error]` | 赤 / 🔴 | エリア内のチャンス、シュート、PK |
| `[CRITICAL]` | 赤 / 🟥 | ゴール |
| `[BREAK]` | シアン | ハーフタイム・給水タイム（recap のトリガー） |

Claude Code のセッションはコードブロック内の ANSI 色をレンダリングできないため、**重大度は絵文字で表示**します。ターミナルの `tail -f` では **ANSI 色**で見えます。

## 仕組み

```
┌─ daemon (バックグラウンド) ───────────────────────────┐
│  ESPN 非公式 API をポーリング (既定 10s)              │
│    → イベント分類: tier(ポーリング速度) + severity     │
│    → claude -p (headless) で流れ系イベントを実況化     │
│    → ~/.worldcup-live-cli/match-{id}.log に追記        │
│  危険時(シュート/PK/ゴール): 3s に短縮 + 即時出力       │
│  静かな区間: ~10s のアンビエント行で無音を排除          │
└───────────────────────────────────────────────────────┘
        │
        ├─ Claude Code セッション (スキルが自動で):
        │    follow が新しい行だけを long-poll で取得し出力
        │    + 10分 / [BREAK] ごとに recap を投稿 (byte-cursor で重複出力を遮断)
        │
        └─ ターミナル: tail -f ~/.worldcup-live-cli/match-{id}.log
```

実況の主体はデーモンであって、あなたのセッションではありません ―― だからセッションを開いたままにする必要はなく、本業に集中できます。危険な場面は遅延ゼロのテンプレートで即座に出力し、流れ系のイベントだけを `claude -p` のバッチで実況化します（実測で `claude -p` 1 回あたり約 11s かかり、3s 周期に追いつけないため）。中継テキストはすべて読み手の言語で出力され、固有名詞（選手名・チーム名）はそのまま残します。

## 設定

`~/.worldcup-live-cli/config.json`（例は `config.example.json`）:

- `ambientIntervalSec` ―― アンビエント行の最小間隔（既定 10 秒、下限 3 秒）
- `pollIntervalSec` / `tier2PollIntervalSec` ―― ポーリング周期（下限 10s / 3s、コードで固定）
- `tier2.lateGameMinute` / `closeScoreDiff` / `cooldownSec` ―― 高速ポーリングへの昇格しきい値
- `narrator.mode`（`auto` / `claude` / `template`）、`model`、`timeoutSec`

## 開発

ソースは TypeScript（`src/`、`scripts/`）で、配布物は esbuild でまとめた単一の `dist/poll.mjs`（ランタイム依存ゼロ）です。

```bash
npm install            # tsx / esbuild / typescript (dev のみ)
npm run build          # scripts/poll.ts → dist/poll.mjs
npm run typecheck

node dist/poll.mjs list
node dist/poll.mjs daemon <eventId> &
node dist/poll.mjs replay <eventId> [--speed <n>] &
tail -f ~/.worldcup-live-cli/match-<eventId>.log
```

スキルを変更したら、`npm run build` で `dist/poll.mjs` を更新してコミットしてください ―― インストールするユーザーはバンドルだけを受け取ります。

## データソースと制限

ESPN の非公式（undocumented）API を使用します。認証は不要ですが、公式サポートも SLA もありません。スキーマはいつ変わるか・遮断されるか分かりませんが、その場合でもデーモンは死なず、生 JSON をサイドカー（`match-{id}.raw.jsonl`）に残して持ちこたえます。

同じ理由で、ポーリング間隔の下限（10s / 3s）はコードに固定されており、設定でこれ以上短くはできません。このツールが人気になってエンドポイントが遮断されれば、全員が損をします。お行儀よく使いましょう。

マイナーな試合では、分刻みのコメンタリーが無く keyEvents のみが提供される場合があります。その場合は中継の密度は下がりますが、ビッグイベントとアンビエントな流れは維持されます。
