# e2e-monitor v2 — 출력 경험 개선 프롬프트

> 2026-06-13, replay 사용 피드백 기반. 루프엔지니어링 프롬프트 — 에이전트에게 그대로 전달.

[큰 그림]
월드컵 경기를 직군 산출물로 위장한 텍스트 중계 데몬(e2e-monitor)을 만들고 있다. v1 데몬과 replay 커맨드는 동작하지만, 중계를 보려면 사용자가 직접 로그 명령어를 쳐야 해서 "스킬 하나 실행하면 알아서 흘러나오는" 경험이 아직 아니다.

[산출물 용도]
bk가 Claude Code 세션 안에서 스킬로 실행해 경기 중계를 실시간으로 지켜보는 도구다. 라이브 중계와 replay가 같은 출력 경험을 공유한다.

[성공 조건]
- 1순위: 스킬 실행 시 Claude Code 세션 안에서 중계가 끊기지 않고 계속 표시된다. 이게 기술적으로 불가능할 때만 2순위 — 중계 로그를 보여주는 화면(tail 등)이 자동으로 떠야 하며, 사용자에게 명령어를 안내하고 끝내면 실패다.
- 중계 텍스트는 전부 한국어다 (코드 식별자·고유명사 제외).
- 같은 내용의 로그가 두 번 출력되지 않는다 — polling 결과가 직전과 동일하면 아무것도 내보내지 않는다.
- ESPN 응답을 그대로 뿌리지 않는다. 의미 있는 변화(득점·카드·교체·경기 상태 전환 등)만 골라 각색해 내보내고, 나머지는 버린다.
- 위 조건이 라이브 중계와 replay 양쪽에 동일하게 적용된다.

이걸 감안해서:
e2e-monitor의 출력 경로만 손봐라 — 스킬 실행 → 중계가 알아서 흘러나오는 경험을 만들고, 중복·노이즈 로그를 걸러내고, 모든 중계 텍스트를 한국어로 바꿔라. 중계 컨텐츠 생성 로직(스킨·각색)은 이 조건을 만족하는 데 필요한 만큼만 건드린다.
